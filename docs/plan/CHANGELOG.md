## 2026-06-24

- **A course is now a grammar (loop closed):** `scripts/course_to_grammar.py` converts a markdown course (`course/<id>.mdx`) into a grammar — each `##` section a lesson, its markdown the `Content`. Converted the **History of Tarot** course → `tarot/history-of-tarot-course/grammar.json` (16 lessons, big ones). It now flows through the whole grammar pipeline — viewable in cards/explorer, forkable, publishable, editable — and renders back as a polished course. To support that, `grammar-course.html` was upgraded to **full markdown** (via `marked`, like our course pages) with editorial typography (blockquotes, lists, headings, images, tables) and renders a `Content` section as **unlabeled lesson prose** — so a converted course reads like a real course (full lessons), not a stack of cards.

- **Cards-viewer header buttons tidied:** removed **Explore More** and the **Read as a course** button; **Get a Reading** and **Copy to drafts** are now both solid **purple + white** and sit side by side (Copy moved up next to Get a Reading and given its text label back — it had been icon-only). Both are recursive.eco actions, so purple matches the redirect convention. The course view is still reached via the eye view-switcher and the contribute page.

- **Grammar-as-course is now a first-class view (the whole flow, built in the tarot repo to cherry-pick into recursive.eco):** the shared eye view-switcher (`view-switcher.js`) gains **Course** — flip any grammar to `grammar-course.html` (each item a lesson), carrying `?src=`. It's reachable from every viewer that shows the eye (timeline / tree / genealogy / print) and the course view itself shows the eye to flip back to Cards / Explorer. The cards viewer (which keeps its view-switching in the header nav) gets a direct **Read as a course** button in its actions row. So the round-trip Cards ↔ Course ↔ Explorer works for any grammar — the exact Eye-dropdown pattern to port into recursive.eco's disabled course view.

- **Card-detail eco buttons → one readable Edit + Play pair:** removed the top "Cast · View · Edit" bar — its gold **Cast** button rendered unreadable (purple-on-gold: the theme's eco-redirect purple rule out-specified the white text), and it duplicated the foot chips. The card detail (cards viewer + Caster) now shows exactly two purple chips: **Edit in recursive.eco** (pencil → the editor opened on that card) and **Play in recursive.eco** (play → the Journal with the deck loaded in the Caster, via `?deckId=`). Each carries its own action icon; the chip's spiral mark was dropped in favour of the pencil/play glyphs.

- **Published the How-to-Contribute grammar to recursive.eco** (via the MCP): created `368974ce-0718-4e9a-9351-bce7b2c053e4`, added the 14 (what × how) items, published it **public + open-to-community**. Mapped it in `_eco_ids.json` (so all 37 repo grammars now open in the app) and turned the contribute page's "Build your own" into a real **Fork this guide ↗** link (`forkId`). Side note: the MCP writes ran clean from Claude Code — the connector approval gate that blocks claude.ai does not apply here, exactly as the diagnosis predicted.

- **Canonical "open in recursive.eco" chip:** standardized the redirect-to-the-app affordance into ONE recognizable pattern — the recursive.eco **spiral mark + "Open in recursive.eco ↗" / "Edit in recursive.eco ↗"** in the platform purple (`#9333ea`), defined once as `.eco-chip` in `theme.css` and applied in the cards viewer + the Caster. Replaces three different looks (the cards' gold pencil/grid "…on recursive.eco →" buttons — a drift from the purple-redirect convention — and the Caster's "View deck ↗" / "Edit card ↗"). Rung 1 of the contribute course now teaches it ("look for the spiral"). **Folder NOT renamed** (investigated `tarot/ → grammars/`): recursive.eco's Cast page hardcodes `…/recursive-tarot/main/tarot/<slug>/grammar.json` for card-detail + "Save to library" import, so a rename would 404 the live cross-platform import — left as-is.

- **Grammar → course (the dogfood victory):** new `viewers/grammar-course.html` renders ANY grammar as a readable, TOC'd **course** — each item becomes a lesson (its `sections` map = the prose, with **bold** / `code` / links rendered), and an "Organize by" dropdown re-chapters the whole course by any metadata dimension. Proven on the new **How to Contribute** grammar (`tarot/how-to-contribute/grammar.json`, generated by `scripts/build_contribute_grammar.py`): 14 items, one per (what × how) way-in, chaptered by `what` or `how`. The contribute page now carries a **"This guide is itself a grammar"** callout with all three reads of the same file — as a course, as the what × how matrix in the Explorer, or build-your-own on recursive.eco. This is the grammar-as-course pattern to port to recursive.eco's course view.

- **Contribute course (Rung 5 — MCP setup):** added an **"Allow the tools"** section that names the single most common reason "the MCP doesn't work" — claude.ai's connector approval gate.
