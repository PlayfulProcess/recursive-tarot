## 1. Architecture (what lives where — single source of truth)

- **Evidence + quotes** live in `research/` dossiers (`decks/`, `people/`, `cards/`),
  format per `research/SCHEMA.md`, sources in `research/bibliography.bib`.
- **Readable catalogue text** lives in the grammars (`tarot/<slug>/grammar.json`) — the
  source of truth for *display*. Each enriched grammar gets a `_research` pointer.
- **People** are their own grammar (`tarot/people-of-tarot/grammar.json`); nodes carry
  `made: [deck ids]` and, where specific, `features_cards: [card ids]`.
- **The meta** (`all-decks-many-lenses`) is generated by `build_meta_grammar.py` — never a
  source of truth. Goal: keep shrinking what only the meta knows, so it becomes a pure
  index/summary, not a second truth. (Maintainer wants its *necessity* to disappear.)

## 2. The per-deck loop (repeat for each deck)

1. **Research** → write/Extend `research/decks/<slug>.md` + `research/cards/<slug>.md`,
   web-verified, every load-bearing claim cited `[@key]` (add keys to bibliography.bib).
2. **People** → ensure each maker/patron/scholar has `research/people/<slug>.md` + a node
   in the people grammar, linked to this deck.
3. **Integrate** → update the deck `grammar.json` (description, per-card sections,
   metadata: order/function/ancestry/derives_from/people). Fix any error the research found.
4. **Validate** → `node scripts/validate-grammar.mjs origin/dev` + a JSON load check +
   `python3 scripts/build_meta_grammar.py` (must stay dangling=0).
5. **Commit + push** (`git push -u origin claude/tarot-deck-accuracy-n4dv2r`). One deck per
   commit where practical. Update this ledger.

Agents: research (steps 1–2) can be fanned out in parallel to sub-agents writing dossier
files on **disjoint paths**; integration (steps 3–5) is done by the orchestrator to keep
grammar edits controlled and validated.

## 5. Guardrails (do not violate)

- Game decks have **no native divinatory meanings** — don't import RWS meanings into them.
- Preserve peculiarities (Etteilla's renumbering, Minchiate's extra trumps) — don't normalise.
- Commit + push frequently (sleep-safe). One logical unit per commit.
- Keep `dangling=0` in the meta build; keep every grammar JSON-valid.
