What this is — and how it feeds recursive.eco

recursive.eco is a place where people make sense together using grammars — symbolic systems like tarot decks, stories, and I Ching books. This repository is the library behind it: each deck is a plain JSON file in tarot/, public and forkable. The app is the window that renders those files as cards, trees, timelines and oracles. The viewers you see here are thin — the data is the point.

A short, honest history of the cards

Tarot began as a card game in 1440s northern Italy — extra trump cards (trionfi) added to an ordinary four-suit pack imported from the Islamic world. For three centuries it was played, not read. Only in 1781 did Antoine Court de Gébelin claim the cards held lost Egyptian wisdom, and Etteilla turned that claim into a cartomancy business. The "ancient esoteric tarot" is itself an 18th-century invention — which makes it no less interesting, only younger than it pretends.

Some traditions hold that an older, hidden esoteric tarot existed before the written record. We can't disprove that — but this repo/project follows the cautious, evidence-first path of historians and primary sources: where the documents are silent, we don't fill the gap with speculation. An unobserved version may yet prove real; we simply don't claim what the evidence can't show. That is the scientific posture, not a denial.

Common misconceptions, corrected

  • "Tarot is ancient Egyptian." No — it's 15th-century Italian, and the Egyptian story dates only to 1781.
  • "The Major Arcana came first." The suits came first (from Mamluk playing cards); the trumps were added to them.
  • "Tarot was always for divination." For ~340 years it was a trick-taking game; cartomancy is the late chapter, not the root.
  • "The Rider-Waite-Smith is the original." It's from 1909 — its illustrated pips borrow from the 1491 Sola Busca, here in the library.
  • "One true meaning per card." Each tradition reads differently; this library keeps each deck in its own tradition rather than forcing one system across all.

How to contribute

It's a public commons. Spot an error, want to add a public-domain deck, or improve a description? Fork the repo, edit the JSON or research, and open a pull request — or just open an issue.
