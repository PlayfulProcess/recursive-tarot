The short version

A game first, a mirror later

Tarot began as a card game in 1440s northern Italy — extra trump cards (trionfi) added to an ordinary four-suit pack imported from the Islamic world. For three centuries it was played, not read. Only in 1781 did Antoine Court de Gébelin claim the cards held lost Egyptian wisdom, and Etteilla turned that claim into a cartomancy business. The "ancient esoteric tarot" is itself an 18th-century invention — which makes it no less interesting, only younger than it pretends.

Some traditions hold that an older, hidden esoteric tarot existed before the written record. We can't disprove that — but this project follows the cautious, evidence-first path of historians and primary sources: where the documents are silent, we don't fill the gap with speculation. Corrected

Common misconceptions

  • "Tarot is ancient Egyptian." Half true, for the wrong reason — the four-suit pack was built on cards from Mamluk Egypt (Cairo). But the trumps that make it tarot are 15th-century Italian, and the "lost Egyptian wisdom" of Thoth is an 18th-century invention (1781), not a pharaonic one.
  • "The Major Arcana came first." The suits came first (from Mamluk playing cards); the trumps were added to them.
  • "Tarot was always for divination." For ~340 years it was a trick-taking game; cartomancy is the late chapter, not the root.
  • "The Rider-Waite-Smith is the original." It's from 1909 — its illustrated pips borrow from the 1491 Sola Busca, here in the library.
  • "One true meaning per card." Each tradition reads differently; this library keeps each deck in its own tradition rather than forcing one system across all.

How it's built

An open library, not an app

recursive.eco is a platform where people make sense together using grammars — symbolic systems like tarot decks, I Ching books, and story frameworks. This repository is the open library: each deck is a plain JSON file in tarot/, public and forkable. The viewers here are thin, backend-free static pages on GitHub Pages — the richness lives in the data and in the app.

It's a public commons. Spot an error, add a public-domain deck, improve a description — fork the repo and open a pull request, or follow the contributor's guide. Every image must be public domain with attribution; every meaning stays in its source tradition.
