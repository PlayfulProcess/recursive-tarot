# Vibe Coding 101

Vibe coding is steering, not typing. You describe the outcome you want in plain language, Claude Code makes the change, and you look at the result and react. The skill is the loop: small asks, quick checks, honest feedback.

## Set up your workspace

Open Claude Code in a folder you own. If it is a fresh idea, ask Claude to make the folder and start a git repo so every step is saved and reversible.

```
mkdir my-project && cd my-project
git init
```

## Describe the outcome, not the code

Tell Claude what you want to exist, not how to write it. Be concrete about the result and the constraints. "Make a single HTML page that shows a clock that ticks, no dependencies" beats "write some JavaScript."

## Look at the result every time

After each change, actually run it. Open the file, refresh the page, read the output. The whole method depends on closing the loop with your own eyes — never accept a change you have not seen work.

## Give feedback in plain words

When something is off, say what you see and what you wanted. "The clock shows the wrong timezone and the font is tiny" is enough. Claude adjusts; you look again. Repeat until it is right.

## Commit when it works

When a step is good, save it. Ask Claude to commit with a clear message. Now you can try bold changes without fear — you can always come back.

```
git add -A && git commit -m "clock ticks, correct timezone"
```

## Push it into the world

When you are proud of it, put it somewhere people can see it. Claude can create the repo, push, and even turn on a live preview. The point of building is sharing.
